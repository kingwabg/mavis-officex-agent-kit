03dispatch 알고리즘

조건 분류 결과 할당
intent ∈ {patch, fix, typo, rename} simple [dev-idle] → tester → deploy
files ≤ 1 && lines ≤ 10 && AGENTS.md 무변경 simple [dev-idle] → tester → deploy
그 외 (schema / 새 도메인 / 큰 리팩) full arch(auto-pass) → [dev-idle] → tester → deploy
큰 feature를 모듈로 분할 가능 parallel arch(auto-pass) → [dev-1 + dev-2 + dev-3] → tester → deploy
※ [dev-idle] = Mavis가 idle한 dev 슬롯 자동 선택. 3 슬롯 동일 풀스택 — 어떤 task든 받음.

05실제 demo — A vs B

A v1 (target 누락)
~6분 FAIL
A v2 (1줄 zero-pad)
~54s ✅
B (P2 children 분리)
~11분 ✅
A v2: producer 54초 (실제 가속화 목표 ~50s 달성) — 1줄 변경이라 typecheck 외 분석 없음.
B: producer 9분 (목표 ~80s 미달) — 새 파일 3개 + import 경로 grep이 메인 cost. verifier PASS 자체는 2분.
가속화는 cycle overhead만 줄이고 절대 작업 시간은 못 줄임.

06알려진 이슈 — cross-plan git diff

증상: 같은 /tmp/sc- 작업트리에 2개 plan 동시 실행 시 verifier의 git diff --stat이 cross-plan 변경 포함 → "scope 위반" 오판.
